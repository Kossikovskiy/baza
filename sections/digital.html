Цифровая безопасность

Аккаунты

Пароли

Один пароль для всего — один замок на все двери.

  • Уникальный пароль для каждого сервиса — без исключений.
  • Надёжный пароль: 12+ символов, разный регистр, цифры, спецсимволы.
  • Хранить в менеджере паролей: Bitwarden (бесплатный, с открытым кодом), 1Password, KeePass.

Проверить, не утекли ли ваши данные: haveibeenpwned.com — введите email и увидите, в каких утечках он засветился.

Ошибки: одинаковые пароли; хранение в заметках или браузере без мастер-пароля; «qwerty123» или дата рождения.

Двухфакторная аутентификация (2FA)

Даже утёкший пароль бесполезен без второго фактора.

СпособНадёжностьЗамечания
SMS-кодСлабаяПерехватывается через подмену SIM или уязвимость SS7
TOTP-приложениеХорошаяGoogle Authenticator, Authy, 2FAS — код живёт 30 секунд
Аппаратный ключ (YubiKey)НаилучшаяФизическое устройство — для максимальной защиты
  1. Включить 2FA везде: почта, банки, Госуслуги, соцсети, рабочие сервисы.
  2. Предпочитать TOTP-приложение вместо SMS.
  3. Сохранить резервные коды отдельно от устройства с 2FA.

Потеряли телефон с TOTP без резервных кодов — доступ к аккаунту может быть утрачен навсегда.

Угрозы

Фишинг

Поддельный сайт визуально неотличим от оригинала. Проверяется только URL.

  • Адрес отправителя: @gosuslugi-help.ru ≠ @gosuslugi.ru.
  • Смотреть на URL в адресной строке, а не на внешний вид страницы.
  • По ссылкам из писем и мессенджеров не переходить — открыть сайт напрямую через браузер.
  • Банк, Госуслуги, налоговая никогда не просят ввести данные карты или пароль по ссылке из письма.

Классическая схема: «Ваш аккаунт заблокирован, перейдите для восстановления». Признаки: паника, срочность, ссылка.

Сомнения — позвонить в организацию по номеру с официального сайта, не из письма.

Обновления

Большинство взломов — через известные уязвимости, для которых патч уже выпущен, но не установлен.

  • Включить автообновления для системы (Windows, macOS, Android, iOS).
  • Обновлять браузер и расширения — самый популярный вектор атак на домашних пользователей.
  • Удалять неиспользуемые приложения: каждое — лишняя точка риска.
  • Устаревшие ОС (Windows 7, Android 6 и ниже) не получают обновления безопасности — это риск, который не убирается настройками.

Отключать обновления «чтобы не мешали» — это выбор между неудобством и уязвимостью.

Публичные Wi-Fi сети

Открытый Wi-Fi — чужая сеть. Трафик до роутера может прослушиваться.

  • Не вводить пароли, данные карт и логины через открытые сети.
  • Использовать VPN: он шифрует весь трафик между устройством и сервером.
  • Без VPN — только HTTPS-сайты (замок в адресной строке).
  • Мобильный интернет (4G/5G) надёжнее публичного Wi-Fi для чувствительных операций.

Банковские операции через открытый Wi-Fi в кафе, аэропорту или гостинице — частый источник проблем.

Резервные копии

Правило 3-2-1

Потеря данных — не вопрос «если», а вопрос «когда».

Правило 3-2-1: 3 копии данных → на 2 разных типах носителей → 1 хранится вне дома или в облаке.

  • Документы и фото → облако (Google Drive, Яндекс Диск, iCloud) + внешний диск.
  • Смартфон → автоматический бэкап в облако в настройках устройства.
  • Пароли → менеджер с облачной синхронизацией; резервные коды 2FA отдельно.

Единственная копия важных файлов — на ноутбуке или внешнем диске рядом с ним. И то, и другое теряется одновременно.
