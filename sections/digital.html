Цифровая безопасность

Пароли: один сервис — один пароль

Повторяющийся пароль — это один замок на все двери. Одна утечка открывает всё.

  • Использовать уникальный пароль для каждого сервиса — без исключений.
  • Надёжный пароль: от 12 символов, буквы разного регистра + цифры + спецсимволы.
  • Хранить в менеджере паролей: Bitwarden (бесплатный, открытый код), 1Password, KeePass.

Проверить, не утекли ли ваши данные: haveibeenpwned.com — ввести email и посмотреть, в каких утечках он фигурирует.

Ошибки: одинаковые пароли; хранение в заметках или незащищённом браузере; пароль вида «qwerty123» или дата рождения.

Двухфакторная аутентификация (2FA)

Даже если пароль утёк — без второго фактора войти в аккаунт невозможно.

Способ 2FAНадёжностьПримечания
SMS-кодСлабаяМожно перехватить через подмену SIM или уязвимость SS7
TOTP-приложениеХорошаяGoogle Authenticator, Authy, 2FAS — код живёт 30 секунд
Аппаратный ключ (YubiKey)НаилучшаяФизическое устройство — для максимальной защиты
  1. Включить 2FA везде: почта, банки, Госуслуги, соцсети, рабочие сервисы.
  2. Предпочитать TOTP-приложение вместо SMS.
  3. При настройке 2FA сохранить резервные коды — в надёжном месте, не на том же устройстве.

Ошибка: потерять телефон с TOTP и не иметь резервных кодов — доступ к аккаунту может быть утерян навсегда.

Фишинг: поддельные письма и сайты

Визуально сайт-подделка может быть неотличима от оригинала — проверяется только URL.

  • Смотреть на адрес отправителя: @gosuslugi-help.ru ≠ @gosuslugi.ru.
  • Проверять URL в адресной строке, а не внешний вид страницы.
  • По ссылкам из писем и мессенджеров не переходить — зайти на сайт напрямую через браузер.
  • Банк, Госуслуги, налоговая никогда не просят ввести данные карты или пароль по ссылке из письма.

Классическая схема: «Ваш аккаунт заблокирован, перейдите для восстановления» — ссылка ведёт на фейк. Признак: паника + срочность + ссылка.

При малейшем сомнении — позвонить в организацию по номеру с официального сайта, не из письма.

Обновления

Большинство взломов происходит через известные уязвимости, для которых патч уже выпущен — но не установлен.

  • Включить автообновления для операционной системы (Windows, macOS, Android, iOS).
  • Обновлять браузер и расширения — самый популярный вектор атак на домашних пользователей.
  • Удалять приложения, которыми не пользуетесь: каждое — лишняя точка риска.
  • Не использовать устаревшие ОС (Windows 7, Android 6 и ниже) — они не получают обновления безопасности вообще.

Ошибка: отключать обновления «чтобы не мешали» или «не ломали систему» — это выбор между неудобством и уязвимостью.

Публичные Wi-Fi сети

Открытый Wi-Fi — чужая сеть. Трафик между вашим устройством и роутером может прослушиваться.

  • Не вводить пароли, данные карт и логины через открытые сети.
  • Использовать VPN: он шифрует весь трафик между устройством и сервером.
  • Если VPN нет — только HTTPS-сайты (замок в адресной строке браузера).
  • Мобильный интернет (4G/5G) — надёжнее публичного Wi-Fi для чувствительных операций.

Ошибка: банковские операции, вход в почту или рабочие системы через открытый Wi-Fi в кафе, аэропорту или гостинице.

Резервные копии: правило 3-2-1

Потеря данных — не вопрос «если», а вопрос «когда». Диски ломаются, телефоны тонут, файлы удаляются.

Правило 3-2-1: 3 копии данных → на 2 разных типах носителей → 1 хранится вне дома или в облаке.

  • Документы, фото → облако (Google Drive, Яндекс Диск, iCloud) + внешний диск.
  • Смартфон → включить автоматический бэкап в облако в настройках устройства.
  • Пароли → менеджер паролей с облачной синхронизацией + резервные коды 2FA в надёжном месте.

Ошибка: единственная копия важных файлов — только на ноутбуке или только на внешнем диске, который лежит рядом.
